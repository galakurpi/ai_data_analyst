## Reasoning Effort Summary

| Step | Function | Reasoning | Why |
|------|----------|-----------|-----|
| SQL Generation | `generate_sql()` | **LOW** | Structured task, follows patterns |
| Analysis Decision | `should_run_deep_analysis()` | **HIGH** | Critical routing decision |

## Why This Matters

1. **Data-Informed Decisions**: We can't decide what analysis to run without seeing the data first
2. **Visualization with Context**: Deep analysis can pass context to visualization generator
3. **Always Get Data**: SQL execution happens regardless of analysis type
4. **Fallback Safety**: If deep analysis fails, we still have the data for simple viz
